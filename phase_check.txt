* **Mục tiêu:** Trích xuất có chọn lọc các bảng dữ liệu cốt lõi từ SQL Server nguồn và lưu trữ an toàn dưới dạng file hiệu năng cao trên Data Lake.

* **Các bước thực hiện:**

3. **Hợp nhất quan hệ thô:** Tiến hành ghép nối (JOIN) nội bộ các bảng danh mục sản phẩm, bẻ phẳng cây địa lý địa chỉ, gộp thực thể cá nhân/cửa hàng thành `full_name` duy nhất cho khách hàng.

## HƯỚNG DẪN CHẠY PIPELINE (End-to-End)

> **Yêu cầu:** Docker Desktop đang chạy, đã clone repo, đã cấu hình file `.env`.

### Bước 0 — Khởi động toàn bộ hạ tầng Docker

```bash
# Vào thư mục project
cd adventureworks-olap-pipeline

# Khởi động tất cả container
docker compose up -d

# Kiểm tra tất cả container đang chạy ổn định
docker ps

```

*Chờ khoảng 60–90 giây để SQL Server khởi động hoàn chỉnh và tự động phục hồi cơ sở dữ liệu nguồn.*

### Bước 1 — Kiểm tra trạng thái khôi phục dữ liệu gốc SQL Server

```bash
docker exec -it sqlserver_source_oltp /opt/mssql-tools18/bin/sqlcmd \
  -S 127.0.0.1 -U sa -P "AdminPassword123!" -C \
  -Q "SELECT name FROM sys.databases;"

```

*Phải thấy tên database `AdventureWorks2022` xuất hiện trong terminal mới chuyển sang bước tiếp theo.*

### Bước 2 — Chạy tầng BRONZE (Cào 26 bảng thô từ SQL Server → MinIO Parquet)

```bash
docker exec -it spark_master_engine /opt/spark/bin/spark-submit /opt/spark/scripts/ingest_to_bronze.py

```

*Kết quả: Toàn bộ 26 bảng thô được đóng gói dạng Parquet nén Snappy lưu tại `s3a://bronze/`.*

### Bước 3 — Chạy tầng SILVER (Làm sạch, xử lý NULL & Phẳng hóa thực thể)

```bash
docker exec -it spark_master_engine /opt/spark/bin/spark-submit /opt/spark/scripts/transform_to_silver.py

```

*Kết quả: Dữ liệu sạch, chuẩn hóa kiểu dữ liệu được xuất ra tại bucket `s3a://silver/`.*

### Bước 4 — Khởi tạo cấu trúc Kho dữ liệu hình sao tầng Gold (Chỉ chạy lần đầu)

Ní mở pgAdmin tại `http://localhost:5050` (Tài khoản lấy từ file `.env`) hoặc chạy câu lệnh DDL trực tiếp thông qua Docker để tạo 12 bảng trống trên Postgres:

```bash
docker exec -i postgres_gold_dw psql -U postgres -d adventureworks_dw < sql/create_gold_tables.sql

```

### Bước 5 — Chạy tầng GOLD (Tính toán Fact/Dim + Chạy thuật toán ML → PostgreSQL)

```bash
docker exec -it spark_master_engine /opt/spark/bin/spark-submit /opt/spark/scripts/load_to_gold.py

```

*Kết quả: Hệ thống 6 Dim + 6 Fact hoàn chỉnh cùng nhãn phân cụm ML được nạp thẳng vào Postgres.*

### Bước 6 — Truy cập Dashboard Nghiệm thu dữ liệu

Mở trình duyệt và truy cập đường dẫn dưới đây để xem Streamlit hiển thị:

```text
http://localhost:8501
```
