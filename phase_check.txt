## Phase 3: Triển khai tầng BRONZE (Data Ingestion Layer)

* **Mục tiêu:** Trích xuất có chọn lọc các bảng dữ liệu cốt lõi từ SQL Server nguồn và lưu trữ an toàn dưới dạng file hiệu năng cao.
* **Các bước thực hiện:**
1. **Đặc điểm lưu trữ:** Tầng này lưu dữ liệu trực tiếp dưới dạng **File Parquet** trong thư mục lưu trữ (`storage/bronze/`), hoàn toàn không cần viết lệnh SQL DDL tạo bảng Database.
2. Viết tập lệnh PySpark (`extract_source_to_bronze.py`) thực hiện kết nối JDBC vào SQL Server nguồn nằm trong Docker mạng nội bộ.
3. Thực hiện quét và trích xuất đúng 14 bảng thuộc phân hệ Sales, Production, Person phục vụ cho bài toán Bán hàng & Kho vận.
4. Ghi trực tiếp ra file Parquet nhằm tận dụng tối đa khả năng nén dạng cột (Columnar Storage) của Parquet để chuẩn bị cho các bước xử lý nặng tiếp theo.

 docker exec -it spark_master_engine /opt/spark/bin/spark-submit /opt/spark/scripts/ingest_to_bronze.py

## Phase 4: Triển khai tầng SILVER (Data Cleaning & Lịch sử SCD Type 2)

* **Mục tiêu:** Làm sạch dữ liệu thô độc lập và thiết lập cơ chế theo dõi lịch sử thay đổi của thực thể.
* **Các bước thực hiện:**
1. Viết tập lệnh PySpark đọc các file Parquet thô từ tầng Bronze lên RAM của cụm tính toán.
2. **Làm sạch chuẩn hóa:** Thực hiện ép kiểu dữ liệu chuẩn (Số tiền về `NUMERIC`, ngày tháng về `TIMESTAMP`), xử lý triệt để các ô dữ liệu trống (`NULL`), loại bỏ các dòng trùng lặp (Deduplication) và đưa các trường chữ về dạng viết thường (Lowercase) chuẩn hóa.
3. **Cấu hình SCD Type 2:** Viết logic so sánh phiên bản dữ liệu cũ và dữ liệu mới cho bảng Sản phẩm và bảng Khách hàng. Nếu phát hiện thay đổi giá bán hoặc địa chỉ, Spark tự động đóng trạng thái dòng dữ liệu cũ (`is_current = False`, cập nhật `end_date`) và chèn thêm một dòng dữ liệu mới (`is_current = True`, `start_date = hiện tại`).
4. Ghi dữ liệu đã dọn dẹp sạch sẽ xuống thư mục `storage/silver/` dưới dạng file Parquet biệt lập (Bảng nào vẫn giữ nguyên bảng đó, chưa tiến hành JOIN bẻ phẳng).

## Phase 5: Triển khai tầng GOLD (Star Schema & Nhúng Machine Learning)

* **Mục tiêu:** Khởi tạo cấu trúc Kho dữ liệu hình sao vật lý trên PostgreSQL, phẳng hóa phân cấp và dán nhãn các mô hình học máy.
* **Các bước thực hiện:**
1. **Viết script DDL SQL (`create_gold_tables.sql`):** Thực thi lệnh trên PostgreSQL đích thông qua pgAdmin/DBeaver để tạo sẵn bộ khung xương rỗng gồm **4 bảng Chiều (Dim)** và **2 bảng Sự kiện (Fact)** chuẩn Star Schema kèm theo các ràng buộc khóa chính/khóa ngoại.
2. Viết tập lệnh PySpark đọc dữ liệu sạch từ tầng Silver lên.
3. **Phẳng hóa dữ liệu (Denormalization):** Thực hiện phép toán `JOIN` quy mô lớn để bẻ phẳng phân cấp sản phẩm (`Product -> Subcategory -> Category`) thành 1 bảng duy nhất và hợp nhất thông tin khách hàng lẻ/doanh nghiệp kèm địa lý địa chỉ.
4. **Nhúng mô hình AI/ML:** Gọi thư viện **Spark MLlib** thực thi thuật toán **K-Means Clustering** trên bộ chỉ số RFM để phân cụm hành vi khách hàng (`customer_segment`), đồng thời chạy thuật toán phân vị tính toán phân hạng doanh thu hàng hóa (`abc_rank_class`).
5. Ghi đè và gối đầu (Upsert) toàn bộ dữ liệu bảng phẳng đã dán nhãn thuật toán hoàn chỉnh vào hệ quản trị PostgreSQL đích thông qua JDBC Connection.

* **Các bước thực hiện:**
1. Viết mã nguồn Python (`app.py`) ứng dụng framework **Streamlit** để thiết lập giao diện web.
2. Sử dụng thư viện `psycopg2` / `SQLAlchemy` kết nối trực tiếp vào schema `gold` trên PostgreSQL đích để kéo dữ liệu bảng phẳng lên thành Pandas DataFrame.
3. Tận dụng cấu trúc phẳng của Star Schema để viết code filter, groupby cực ngắn gọn trên RAM, loại bỏ hoàn toàn các phép Join phức tạp gây chậm app.

## HƯỚNG DẪN CHẠY PIPELINE (End-to-End)

> **Yêu cầu:** Docker Desktop đang chạy, đã clone repo, đã có file `.env` (copy từ `.env.example` nếu chưa có).

### Bước 0 — Khởi động toàn bộ hạ tầng Docker

```bash
# Vào thư mục project
cd adventureworks-olap-pipeline

# Khởi động tất cả container (lần đầu sẽ pull image, mất vài phút)
docker-compose up -d

# Kiểm tra tất cả container đang chạy
docker-compose ps
```

> Chờ khoảng 60–90 giây để SQL Server khởi động và tự restore AdventureWorks2022.bak.

### Bước 1 — Kiểm tra SQL Server đã restore xong chưa

```bash
docker exec -it sqlserver_source_oltp /opt/mssql-tools18/bin/sqlcmd \
  -S 127.0.0.1 -U sa -P "AdminPassword123!" -C \
  -Q "SELECT name FROM sys.databases;"
```

> Phải thấy `AdventureWorks2022` trong danh sách mới tiếp tục.

### Bước 2 — Chạy tầng BRONZE (Ingest từ SQL Server → MinIO/Parquet)

```bash
docker exec -it spark_master_engine /opt/spark/bin/spark-submit \
  --master local[*] \
  --jars /opt/spark/custom_jars/mssql-jdbc-12.4.2.jre11.jar,\
/opt/spark/custom_jars/hadoop-aws-3.3.4.jar,\
/opt/spark/custom_jars/aws-java-sdk-bundle-1.12.262.jar \
  /opt/spark/scripts/ingest_to_bronze.py
```

> Kết quả: 14 bảng được lưu dạng Parquet vào MinIO bucket `bronze/`.

### Bước 3 — Chạy tầng SILVER (Làm sạch + SCD Type 2)

```bash
docker exec -it spark_master_engine /opt/spark/bin/spark-submit \
  --master local[*] \
  --jars /opt/spark/custom_jars/mssql-jdbc-12.4.2.jre11.jar,\
/opt/spark/custom_jars/hadoop-aws-3.3.4.jar,\
/opt/spark/custom_jars/aws-java-sdk-bundle-1.12.262.jar \
  /opt/spark/scripts/transform_to_silver.py
```

> Kết quả: Dữ liệu sạch + SCD Type 2 lưu vào MinIO bucket `silver/`.

### Bước 4 — Tạo bảng Gold trên PostgreSQL (chỉ chạy lần đầu)

```bash
# Kết nối pgAdmin tại http://localhost:5050
# Email: admin@admin.com | Password: adminpassword
# Thêm server PostgreSQL: host=postgres_dw, port=5432, user=gold_user, pass=adminpassword

# Hoặc chạy DDL trực tiếp qua docker:
docker exec -i postgres_dw psql -U gold_user -d gold_dw \
  < sql/create_gold_tables.sql
```

### Bước 5 — Chạy tầng GOLD (Star Schema + ML Clustering → PostgreSQL)

```bash
docker exec -it spark_master_engine /opt/spark/bin/spark-submit \
  --master local[*] \
  --jars /opt/spark/custom_jars/mssql-jdbc-12.4.2.jre11.jar,\
/opt/spark/custom_jars/hadoop-aws-3.3.4.jar,\
/opt/spark/custom_jars/aws-java-sdk-bundle-1.12.262.jar,\
/opt/spark/custom_jars/postgresql-42.7.3.jar \
  /opt/spark/scripts/load_to_gold.py
```

> Kết quả: Dim/Fact tables + K-Means segment được upsert vào PostgreSQL schema `gold`.
